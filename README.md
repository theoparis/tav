# tav

`tav` is an experimental VCS wrapper inspired by `jj`.

It layers a lightweight change graph on top of an existing colocated Git repository using `github.com/go-git/go-git/v6`.

## Current commands

- `tav init`: wraps the existing Git repo, records `git_head()` in `.tav/state.json`, creates an empty working change, and detaches `HEAD` at the current commit.
- `tav status`: shows the current Tav change and a summarized working tree status.
- `tav log [-r <revset>]`: prints a simple tree-like change log using `◉/◆` markers, relative timestamps, change metadata, branch labels, and `git_head()`.
  - `<revset>` supports a single change (`abcd1234`), or a range (`A..B`, `..B`, `A..`). Special refs: `@` (current), `@-` (current parent).
  - Before rendering, Tav reconciles state with Git `HEAD` so commits created via plain `git commit` are imported into Tav state.
- `tav new [OPTIONS] [REVSETS]...`: jj-like new change creation.
  - Parent selection:
    - positional `REVSETS` (default: `@`)
    - aliases `-o <REVSETS>` and `-r <REVSETS>`
  - `-m, --message <MESSAGE>`: set description on the new change
  - `--no-edit`: create but keep editing current change
  - `-A, --insert-after <REVSETS>`: insert after target(s)
  - `-B, --insert-before <REVSETS>`: insert before target(s)
- `tav describe <message>`: sets the description/message for the current working change.
- `tav commit [message]`: commits the current working change to Git via `go-git`, then creates a fresh empty working change on top.
- `tav squash`: jj-like squash flags (core subset):
  - `-r, --revision <REVSET>`: squash revision into its parent (default source: `@`)
  - `-f, --from <REVSETS>`: source revision (currently one source supported)
  - `-t, --into <REVSET>`: destination revision (default: source parent)
  - `-m, --message <MESSAGE>`: override destination description
  - `-u, --use-destination-message`: keep destination description as-is
  - `-k, --keep-emptied`: keep source change node instead of abandoning it (uncommitted sources only)
  - Experimental destination placement:
    - `-o, --onto <REVSETS>`
    - `-A, --insert-after <REVSETS>`
    - `-B, --insert-before <REVSETS>`
- `tav rebase`: jj-style selector and destination flags:
  - Select revisions with one of:
    - `--source/-s <change>` (change and descendants)
    - `--branch/-b <change>` (branch-style, defaults to `-b @`)
    - `--revisions/-r <a,b,...>` (exact revisions only)
  - Choose destination with one of:
    - `--onto/-o <change>`
    - `--insert-after/-A <change>`
    - `--insert-before/-B <change>`
- `tav op log`: prints Tav operation history from `.tav/oplog.jsonl`.
- `tav push`:
  - `--remote <REMOTE>`: push remote (default: `origin`)
  - `-r <REVSETS>`: push selected committed changes via existing local branches
  - `-c <REVSETS>`: push selected committed changes to autogenerated remote branches (`tav/<change-id-prefix>`)

## Quick start

```bash
go install github.com/theoparis/tav@main

tav init
tav describe "feat: initial change"
tav commit
tav new
tav describe "wip: follow-up"
tav new -m "next step" --no-edit @
tav new -A @-
tav rebase <change-id>
tav squash
tav squash -r @-
tav squash -f @ -t @- -m "squashed message"
tav op log
tav log -r ..@-
tav rebase -b @ -o @-
tav push --remote origin -r @
tav push -c @-
```

Run inside a directory that is already a Git repository.

If the Git repository has no commits yet, `tav init` initializes a root working change without a wrapped `git_head()` commit.
`tav init` also ensures `.tav/` is listed in `.gitignore`.

## Current limitations

- `tav rebase` currently operates on Tav working changes (non-materialized changes) only.
- `tav squash` supports committed changes in Tav's virtual graph; when both sides are committed, the squash target must be an ancestor of the current change.
